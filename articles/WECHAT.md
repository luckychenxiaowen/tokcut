# 我花了 6 小时，给国产大模型装上了一个「省 token 引擎」

**一个不修改任何代码、就能让你的 LLM API 开销直接砍掉 40% 的开源工具。**

---

事情的起因很偶然。

周末刷 GitHub，看到一个叫 Caveman 的项目。它的做法非常简单粗暴——往 AI 的系统提示词里加一段指令，要求模型「用最简练的方式回答，省掉所有客套话」。就这么一个操作，在 Claude Code 里实测省了 65% 的输出 token。

65%。如果你每个月在大模型 API 上的开销是几千甚至上万，这个数字意味着什么，不言自明。

但 Caveman 有个局限：它只做了输出端的压缩，而且只针对 Claude Code 场景。能不能把这个思路放大，做成一个通用的、覆盖更多模型和更多压缩维度的工具？

这种念头一旦冒出来，就跟中毒了一样，不干完睡不着。

---

## 三层压缩，同时发力

我花了不少时间把市面上各种 token 节省方案翻了个遍。多是真的多——prompt 压缩、语义缓存、KV cache 压缩、本地路由、CoT 压缩……十几种，每种都有人吹。

但说实话，大多数要么绑定了特定模型，要么实施起来坑太大。真正适合扒下来做成通用轮子的，一只手数得过来。

最终我选定了三个方向：

**第一层：输出风格压缩。** 继承了 Caveman 的核心思路，但做了两件事让它更普适。一是分了三个档位——lite、full、ultra，用户可以根据场景选。二是在压缩前后加了「技术内容保护层」，代码块、URL、版本号这些信息会被临时替换成占位符，压缩完再换回来，确保关键信息不丢失。

**第二层：输入语义压缩。** 用户的提问里有很多冗余——「请问」「能不能帮我」「麻烦一下」这些词，对于 AI 理解意图其实没什么帮助，但都会计入 token。去除它们，再加上相邻重复行的合并，能在不改变语义的前提下省掉 20%-40% 的输入 token。

**第三层：语义缓存。** 这是三个模块里 ROI 最高的。原理是用一个轻量级的语义模型把用户请求转成向量，然后跟历史上处理过的请求比较。如果相似度超过某个阈值——比如 0.95——就直接返回之前的结果，完全不走 LLM。对于高频重复问题，这一层能省掉几乎全部 token。

---

## 实测数据

我在 DeepSeek-V3、GLM-4、MiniMax-M2.5、腾讯混元、Kimi 五个模型上，各跑了五类任务，一共 25 个测试用例。

结果比预期要好：

| 模型 | 节省率 |
|------|--------|
| MiniMax-M2.5 | 44% |
| DeepSeek-V3 | 42% |
| GLM-4 | 41% |
| 腾讯混元 | 40% |
| Kimi | 38% |

平均下来，五个模型的总节省率是 **41%**。

而且这只是输出压缩+缓存的保守配置。输入压缩我都没开。要是把 safe 模式的输入压缩也挂上，还能再抠出来一些。

质量方面我也专门测了——代码块、数字、URL 这些关键信息有 ContentProtector 兜着，压缩完原样还回去。所以不需要担心「省钱省到把答案搞坏了」。

## 六个小时，从零到开源

这个项目的开发方式本身，可能比代码更有意思。

坦白说，33 个文件、两千多行代码，我实际手敲的不到十分之一。流程简化下来就四步：

先用 DeepSeek 把设计文档写出来。模块拆几个、核心逻辑怎么走、API 长什么样——这部分主要是我在做判断，AI 帮我把思路结构化。

然后把设计文档丢给 CodeBuddy。它直接吐出全套工程文件——源码、配置、Docker、CI/CD、中英文文档，一个不少。

代码出来之后就跑测试、改 bug、调参数。来回折腾了好几轮，这部分最耗神。

最后补社区治理的东西——贡献指南、行为准则、安全策略、Issue 模板。说实话比写代码还累，但开源项目没这些不行。

全程六个多小时。从脑子里一个朦胧的想法，到 GitHub 上能跑的东西。

要说 AI 完全替你写了编程，那绝对是吹牛。设计该怎么定、接口怎么权衡、参数怎么调——这些决策还是自己的脑子。但 AI 确实把「我知道要写什么，就是懒得打出来」那部分工作给省了。这一省，省出来的时间是以前没法想象的。

## 怎么用

最简单的用法：

```bash
# 1. 安装
pip install tokcut

# 2. 启动代理
python -m tokcut.server

# 3. 把客户端的 API 地址改掉
# base_url 从 "https://api.deepseek.com/v1"
# 改成 "http://localhost:8800/v1"
```

一行代码都不用动。所有 OpenAI 兼容的模型——DeepSeek、GLM、Kimi、混元、MiniMax——都能直接接入。

如果你想知道每个请求到底省了多少，每个响应里都会带一个 `tokcut` 字段，输入前、输入后、输出量，一目了然。
