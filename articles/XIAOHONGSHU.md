# 🤯 刷到一个 GitHub 项目，我直接熬夜肝了个开源工具出来

周末刷 GitHub Trending，看到一个叫 Caveman 的项目🦴

这玩意做的事情巨简单——给 AI 的 system prompt 里塞一句话：「说话别整那些虚的，直接给答案」

结果呢？Claude 的输出 token **直接省了 65%**⚠️

我看到这个数字的时候整个人都不好了🤯

## 💡 脑子里开始跑马灯

「65% 啊……如果每个月 API 开销一万块，这就省了六千五」

「但它只管输出不管输入，也没有缓存功能」

「要不……我自己做一个更狠的？」

---

## 🛠 说干就干

思路不复杂：
1. 让 AI 输出尽量简洁（学 Caveman）
2. 把用户输入里那些「请问」「麻烦一下」全删掉
3. 如果两个问题问得差不多，直接返回上次的答案，不再调 AI

这三个招叠在一起，叫它 **tokcut**（token 剪刀的意思✂️）

---

## 🤖 全程用 AI 写的代码

说出来你可能不信，这 33 个文件我基本没手写几行

步骤是这样的👇

1. 先用 DeepSeek 把设计文档写出来——模块怎么拆、接口怎么定
2. 然后把设计文档扔给 CodeBuddy，让它把全部代码吐出来
3. 修修 bug、调调参数、跑跑测试
4. 最后补齐文档，推到 GitHub

前后大概一个下午加一个晚上⏱

放以前，光搭项目框架就得两天

---

## 📊 效果怎么样

我在 5 个国产大模型上跑了一遍对比测试

- DeepSeek：省 **42%** ✨
- GLM-4：省 **41%** ✨
- MiniMax：省 **44%** ✨
- 腾讯混元：省 **40%** ✨
- Kimi：省 **38%** ✨

平均下来 **省了 41% 的 token**

而且回答质量几乎没降！代码、链接、数字这些技术内容都被自动保护了，不会被误删

---

## 🎁 怎么用

只需要把代码里的 API 地址改一下就行：

```python
# 之前这样调 DeepSeek
client = OpenAI(base_url="https://api.deepseek.com/v1")

# 现在换成 tokcut
client = OpenAI(base_url="http://localhost:8800/v1")
```

其他代码**一行都不用改**
