# Token 节省效果对比报告

> DeepSeek-V3 真实 API 对比测试 —— 使用 system prompt 注入压缩指令 vs 普通调用

## 测试信息

- **测试模型**: DeepSeek-V3 (`deepseek-chat`)
- **测试框架**: 5 类任务（代码修复、文本总结、知识问答、翻译、对话）
- **测试方法**: 同一 prompt 分别用普通模式、full 压缩、ultra 压缩各调用一次
- **压缩方式**: 在 system prompt 中注入 Caveman 风格简洁指令（与 tokcut 核心逻辑完全一致）
- **测试时间**: 2025-05-02

## 测试任务

| # | 类别 | Prompt |
|---|------|--------|
| 1 | 代码修复 | React.memo 失效原因及修复代码 |
| 2 | 文本总结 | 产品规划会议纪要总结 |
| 3 | 知识问答 | RLHF 核心原理解释 |
| 4 | 翻译任务 | AI 伦理相关英文翻译中文 |
| 5 | 对话任务 | 客服订单发货延迟回复 |

## 详细结果

### Full 级别压缩

| 任务 | 普通模式 | Full 压缩 | 节省 Token | 节省率 |
|------|---------|----------|-----------|--------|
| 代码修复 | 1,243 | 395 | 848 | **68.2%** |
| 文本总结 | 130 | 131 | -1 | -0.8% |
| 知识问答 | 1,506 | 224 | 1,282 | **85.1%** |
| 翻译任务 | 45 | 91 | -46 | — |
| 对话任务 | 135 | 120 | 15 | 11.1% |
| **总计** | **3,059** | **961** | **2,098** | **68.6%** |

> 输出 Token 节省：2,901 → 568（**80.4%**）

### Ultra 级别压缩

| 任务 | 普通模式 | Ultra 压缩 | 节省 Token | 节省率 |
|------|---------|-----------|-----------|--------|
| 代码修复 | 1,243 | 173 | 1,070 | **86.1%** |
| 文本总结 | 130 | 107 | 23 | 17.7% |
| 知识问答 | 1,506 | 149 | 1,357 | **90.1%** |
| 翻译任务 | 45 | 86 | -41 | — |
| 对话任务 | 135 | 115 | 20 | 14.8% |
| **总计** | **3,059** | **630** | **2,429** | **79.4%** |

> 输出 Token 节省：2,901 → 267（**90.8%**）

## 关键发现

### 1. 长文本回答节省效果极佳

代码修复和知识问答这类需要详细解释的任务，压缩效果最为显著：
- 知识问答（RLHF 解释）：普通模式输出 1,488 tokens 的详细教程，full 压缩后仅 159 tokens，**节省 85%** 仍覆盖了所有核心概念
- 代码修复：1,208 → 313 tokens，**节省 74.1%** 输出，仍包含完整的原因分析和代码示例

### 2. 短文本场景需注意

翻译任务（原文仅 16 tokens 输出）中，压缩指令本身消耗的 input token（+47 tokens）超过了节省的输出，导致总 token 反而增加。对于输出量 < 50 tokens 的短任务，**不建议开启压缩**。

### 3. 对话类任务效果温和

客服回复场景（135 → 120 tokens，节省 11%），因为对话本身需要一定的礼貌元素，过度压缩反而影响体验。建议使用 `lite` 级别。

### 4. 质量保持

| 检查项 | 结果 |
|--------|------|
| 技术信息完整性（代码示例、术语） | ✅ 全部保留 |
| 关键事实准确性 | ✅ 无错误 |
| 中文自然度 | ✅ 可接受（full 级别稍显生硬） |

## 综合结论

| 模式 | 总节省率 | 输出节省率 | 适用场景 |
|------|---------|-----------|---------|
| **Full** | **68.6%** | **80.4%** | 技术问答、代码、长文本——推荐日常使用 |
| **Ultra** | **79.4%** | **90.8%** | 机器对机器、批量处理、极简需求 |
| 无压缩 | 0% | 0% | 翻译、短对话、需高质量文案 |

> **建议策略**: 对大多数请求使用 `full` 级别（平均省 68.6%），通过 HTTP Header 按需切换为无压缩模式处理翻译和短对话。
